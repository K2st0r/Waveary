# Commercial Use And Brand Licensing

This page explains the practical boundary between:

- open-source code use
- official Waveary branding
- official visual asset use
- custom commercial cooperation

It is a repository policy summary, not legal advice.

## Short Version

- The source code in this repository is licensed under [Apache License 2.0](../LICENSE).
- The official `Waveary` / `回响之境` name, logos, portrait cards, illustrations, and official promotional visuals are reserved separately.
- You may use the code under Apache 2.0.
- You may not present your product as official Waveary or use the official brand assets without permission.
- If you want official branding, official visual assets, co-branded distribution, or a commercial partnership, you need prior written permission.

## What Personal Use Means Here

For this repository, "personal use" usually means things like:

- running Waveary locally for yourself
- studying the code
- testing model connections
- building private experiments
- modifying the code for learning or internal evaluation

Those activities are governed by Apache 2.0 on the code side.

## What You Can Do Under The Code License

Under Apache 2.0, you may generally:

- use the code
- modify the code
- redistribute modified code
- use the code in commercial software

But Apache 2.0 does not give you trademark rights.

That means code permission is not the same thing as permission to use the official Waveary identity.

## What Requires Rebranding

If you want to build a product, hosted service, packaged app, or paid offering based on this code, you should rebrand unless you have permission to use the official identity.

That means you should not ship it as:

- `Waveary`
- `回响之境`
- an "official Waveary" product
- a visually confusing near-copy of the official brand presentation

## What Requires Prior Written Permission

You need prior written permission if you want to do things like:

- use the official `Waveary` or `回响之境` name in a shipped product or service
- use the official logos
- use the official portrait cards or official promotional illustrations
- use the official GitHub presentation, screenshots, or visual identity as your product branding
- distribute a paid or hosted offering presented as official Waveary
- imply endorsement, partnership, certification, or official status
- request co-branding, OEM-style packaging, or branded commercial distribution

## Common Scenarios

### 1. I want to run it for myself

That is normally fine under the code license.

### 2. I want to fork it and change it

That is normally fine under the code license.

If you distribute it publicly, do not present it as the official Waveary project unless you have permission.

### 3. I want to use the code in a company product

The code license may allow that.

But you should remove the official Waveary branding unless you have separate written permission for brand use.

### 4. I want to sell hosting or packaged builds

Code use may still be possible under Apache 2.0.

Official naming, logos, portrait cards, and official visual assets still require permission.

### 5. I want to keep the official name and visuals

That is a brand-license conversation, not a default code-license right.

## Recommended Safe Path For Builders

If you want the lowest-risk path:

1. Use the code under Apache 2.0.
2. Replace the project name, logo, portraits, and official visuals with your own.
3. Keep attribution required by the code license.
4. Do not imply official affiliation.
