# 商业使用与品牌授权说明

本文用来说明以下几件事之间的边界：

- 开源代码如何使用
- 官方 `Waveary / 回响之境` 品牌如何使用
- 官方视觉素材如何使用
- 商业合作或授权应如何理解

这是一份仓库侧说明，不构成法律意见。

## 一句话版本

- 本仓库源代码遵循 [Apache License 2.0](../LICENSE)。
- 官方 `Waveary` / `回响之境` 名称、Logo、人物卡、插画、宣传视觉等，单独保留权利。
- 你可以按 Apache 2.0 使用代码。
- 你不能未经许可把你的产品包装成官方 Waveary，也不能直接使用官方品牌素材。
- 如果你希望使用官方名称、官方视觉、联名分发、官方合作或其他商业授权，必须事先取得书面许可。

## 这里所说的个人使用

这里的“个人使用”通常包括：

- 你自己在本地运行 Waveary
- 学习、阅读、研究代码
- 测试模型接入
- 做私有实验
- 作为个人或内部评估用途修改代码

这些行为主要受代码许可证约束。

## 代码许可证允许你做什么

在 Apache 2.0 下，通常可以：

- 使用代码
- 修改代码
- 分发修改后的代码
- 在商业软件里使用代码

但 Apache 2.0 不授予商标权，也不自动授予官方品牌权。

也就是说：

代码可用，不等于官方品牌可用。

## 什么时候应该改名改视觉

如果你打算基于这套代码做产品、托管服务、打包客户端、收费服务，最稳妥的方式是：

- 换名字
- 换 Logo
- 换人物卡
- 换官方视觉素材

除非你已经拿到官方品牌授权。

## 哪些情况需要事先书面许可

以下这类情况需要事先书面许可：

- 在对外分发的产品或服务里使用 `Waveary` 或 `回响之境`
- 使用官方 Logo
- 使用官方人物卡、官方插画、官方宣传图
- 使用官方 GitHub 页面、截图或视觉识别系统作为你的产品品牌
- 以“官方 Waveary”名义提供收费托管、收费打包或收费服务
- 暗示官方背书、官方合作、官方认证或官方身份
- 希望做联名、贴牌、联合分发或其他品牌合作

## 常见场景

### 1. 我只是自己跑着用

一般没有问题，按代码许可证理解即可。

### 2. 我想 Fork 后自己改

一般没有问题，代码层面可行。

但如果你要公开发布，请不要继续冒充官方 Waveary。

### 3. 我想把代码接到公司产品里

代码许可证本身可能允许。

但如果继续使用官方品牌、Logo、人物卡、官方视觉，就不行，除非另行授权。

### 4. 我想卖托管服务或打包版本

代码层面未必一定禁止。

但只要涉及官方名称、官方 Logo、官方人物卡、官方视觉，仍然需要单独授权。

### 5. 我就想保留官方名字和形象

这已经不是默认代码权利，而是品牌授权问题。

## 对开发者最稳妥的做法

如果你想最低风险地使用本项目：

1. 按 Apache 2.0 使用代码。
2. 自己替换名称、Logo、人物卡和官方视觉素材。
3. 保留代码许可证要求的归属与声明。
4. 不要暗示自己是官方版本。
